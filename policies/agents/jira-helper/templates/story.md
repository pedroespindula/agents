A História do usuário deve seguir o formato "Eu, como [papel do usuário], gostaria de [o que o usuário quer fazer] para [o benefício esperado]." A Definição de pronto deve detalhar os critérios de aceitação.

```Markdown
# Contexto

[Explicação do problema ou cenário que a história do usuário aborda.]

# História do usuário

Eu, como [papel do usuário], gostaria de [funcionalidade] para [benefício].

# Definição de pronto

- [Critério de aceitação 1]
- [Critério de aceitação 2]

```
