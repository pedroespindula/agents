## What I do

- Criar e manter READMEs de projetos
- Documentar APIs e endpoints
- Escrever guias de uso e tutoriais
- Manter changelogs atualizados
- Criar runbooks operacionais
- Revisar e melhorar documentacao existente

## When to use me

Use this skill when you need to create or improve technical documentation. This is especially helpful when:

- Creating a new project README
- Documenting API endpoints
- Writing changelogs following Keep a Changelog format
- Creating operational runbooks
- Reviewing existing documentation for clarity and completeness

## How I work

I follow documentation best practices:

1. **Clareza**: Documentacao deve ser compreensivel para o publico-alvo
2. **Atualidade**: Documentacao desatualizada e pior que nenhuma documentacao
3. **Objetividade**: Seja direto e evite redundancias
4. **Padronizacao**: Siga templates e convencoes estabelecidas
