# Como Criar Documentação Técnica

Este documento descreve o processo para criação de documentação técnica geral.

## Fluxo de Criação

### 1. Planejamento

Antes de escrever:

- Defina o público-alvo principal
- Identifique o objetivo da documentação
- Liste os tópicos a serem cobertos
- Verifique documentação existente relacionada

### 2. Pesquisa

Colete informações:

- Leia o código fonte relevante
- Converse com especialistas (se disponível)
- Teste funcionalidades você mesmo
- Revise issues e PRs relacionados

### 3. Estruturação

Organize o conteúdo:

- Crie outline com seções principais
- Ordene do mais importante ao menos
- Identifique onde usar exemplos
- Planeje elementos visuais (diagramas, screenshots)

### 4. Escrita

Produza o conteúdo:

- Escreva um rascunho completo primeiro
- Não se preocupe com perfeição inicial
- Inclua todos os exemplos planejados
- Adicione links e referências

### 5. Revisão

Refine o documento:

- Leia em voz alta para verificar fluidez
- Verifique precisão técnica
- Simplifique onde possível
- Corrija erros de gramática e ortografia

### 6. Validação

Confirme qualidade:

- Teste todos os comandos/exemplos
- Verifique se links funcionam
- Peça revisão de outra pessoa se possível
- Confirme que atende o objetivo inicial

## Tipos de Documentação

### Conceitual (O quê)

- Explica conceitos e arquitetura
- Foco em entendimento
- Usa diagramas e analogias
- Exemplo: "O que é o sistema X"

### Procedural (Como)

- Guia passo a passo
- Foco em execução
- Usa comandos e screenshots
- Exemplo: "Como configurar Y"

### Referência (Detalhes)

- Informação completa e precisa
- Foco em consulta
- Usa tabelas e listas
- Exemplo: "API Reference"

### Tutorial (Aprendizado)

- Ensina através de exemplo
- Foco em aprendizado progressivo
- Usa projeto prático
- Exemplo: "Criando seu primeiro Z"

## Checklist de Documentação

- [ ] Objetivo claramente definido
- [ ] Público-alvo identificado
- [ ] Estrutura lógica e organizada
- [ ] Exemplos práticos incluídos
- [ ] Comandos/código testados
- [ ] Links verificados
- [ ] Revisão ortográfica feita
- [ ] Formatação consistente
